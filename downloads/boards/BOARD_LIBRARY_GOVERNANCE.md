# Board Library Governance

## Versioning
- Semantic version per board: `major.minor.patch`.
- Breaking layout/label change increments `major`.
- Additive label updates increment `minor`.
- Typo-only fix increments `patch`.

## Release cadence
- Wave 1: 6 core boards.
- Wave 2: expand to 12 boards after validation pass.

## Review ownership
- SLP review: communicative intent, linguistic appropriateness.
- AT review: access, target size, motor usability, portability.

## Deprecation
- Keep prior version for one release cycle.
- Mark old file in changelog as deprecated.
- Remove deprecated files after next stable cycle.

## Bilingual sync
- Keep EN/RU board tables aligned by row and file mapping.
- Any board add/remove must update both pages in the same change.
